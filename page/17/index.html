SVN服务端数据定时备份脚本

此脚本用于在SVN 服务器上，用户通过SVN 客户端提交后，服务端利用hooks脚本自动更新检出数据的备份！

#!/bin/bash 

#Author  :elain                                                                                                          

#Ctime   :20110805                                                                                                        



Bak_Dir=/data/svnbak                                                                                               

Svn_Dir=/data/svndata

Svn_data=/data/svnbak/svndata_`date +%Y%m%d`.tar.gz

MailList=elain2012@hotmail.com

Info : 每周备份一次svn数据

/bin/tar zcf $Svn_data /data/svndata

Info : 邮件正文

echo -e &quot;==========`date +%Y年%m月%d日` SVN 备份数据==========&quot; >$LogFile

echo -e &quot;SVN备份文件:\nsvndata_`date +%Y%m%d`.tar.gz \n大小:`du -sh $Svn_data |awk &#039;{print $1}&#039;`&quot; >>$LogFile

echo -e &quot;==============by:elain www.elain.org=================\n详情参见附件!&quot; >>$LogFile

Info : 发送邮件

/usr/bin/sendEmail -f admin@elain.org -t $MailList -s mail.elain.org -xu &#039;admin@elain.org&#039; -xp &#039;elain123we&#039; -u &quot;`date +%Y年%m月%d日` SVN备份数据&quot; -m &quot;`cat $LogFile`&quot; -a $Svn_data

Info : 删除七天前的备份

find $Bak_Dir -mtime +7 | xargs rm -rf

多服务器访问日志合并脚本

环境：CentOS 5.5 x64

四台WEB web01 web02 web03 web04

一台日志服务器  log01

需求：

每天定时把四台WEB上的NGINX 访问日志合并后传入给awstats系统分析处理

Info : merge log for many server log

# Author : dingtm                                                                                                    

# CTime  : 2011.03.01

DAY=date -d &#039;-1 day&#039; +%Y/%m/%d 

mkdir -p /data/logs/$DAY/merge_logs/   #建立合并后的日志文件存放目录

OldLog=/data/logs/$DAY #各服务器的日志文件，以日期为目录，各服务器主机名为子目录

NewLog=/data/logs/$DAY/merge_logs

for log in access_ebook.elain.org.log access_www.elain.org.log access_blog.elain.org.log access_books.elain.org.log access_bbs.elain.org.log

  do

    sort -m -t &quot; &quot; -k 4 -o  $NewLog/$log $OldLog/web01/$log $OldLog/web02/$log $OldLog/web03$log $OldLog/web04/$log

done

[转]袁萌：我国普及Linux的回顾与展望

在我们国内，普及Linux的历程时间已经不短了，但是，收效甚微。这是什么道理呢?

近日，在国外围绕Linux诞生20周年的纪念活动逐渐进入高潮，反观我们国内，显得冷冷清清，没有什么动静。现在，世界Linux发展的历史进程 已经清楚证明：GNU/Linux的社区版位居第一(即“为王”)，而不是企业版遥遥领先。比如，根据Linux发行版世界排行榜网站近3个月的统计数 据，GNU/Linux的Mint发行版位居第一，Ubuntu退居第二。俄罗斯的ALT发行版位居第81位，而我们国内的深度(Deepin)发行版则 升至全球第62位(注：国内其他Linux发行版，世界发行版排行榜网站上无名次)。

根据Linux世界排行榜上的官方定义：“Linux Deepin，原名Hiweed GNU/Linux，是一个基于Ubuntu的中文Linux发行版，口号是“免除新手痛苦、节约老手时间”，目标是用起来顺畅、舒服。从10.12版本 开始，采用了GNOME 桌面系统、Yong输入法、永中Office、永中宋体、文泉驿微米黑。”回顾国内Linux发展史，蓝点、冲浪、新华科技与共创开源都相继不见了。为什 么深度Linux发行版能够脱颖而出呢?其根本原因就在于：围绕着它的开发社区一心向往GNU/Linux大方向，专注于Linux桌面研发不转向。

有人说，支持深度Linux是偏心，甚至是别有用心。我说不是。考察Linux发行版，首先应该是国际的，然后才是国内的，而且，必须是由开发社区 驱动的。根据这个选择原则，我们的目光只能放在深度Linux发行版身上。回顾过去，我们曾经有过Linux的纸质出版物(定期刊物)，现在已经全部“清 零”。在我们国内，没有权威性的Linux网站，也没有Linux的官方声音，环顾四周，Linux边界模糊不清。
